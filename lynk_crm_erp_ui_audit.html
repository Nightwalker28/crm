Critical gaps — address first
No real dashboard / home screenCritical
The dashboard home route immediately redirects to the first accessible module. Every major CRM/ERP (Salesforce, HubSpot, SAP, Zoho) opens to a summary view with KPIs, activity feed, and quick actions. Users landing on a raw table list lose immediate context.
Zero record relationship visualizationCritical
Contact → Organization → Opportunity → IO exists in the DB but there is no visual relationship panel on any detail page. Users cannot see "this contact has 3 open opportunities worth $120k" in one glance. This is the core value prop of a CRM detail page.
No pipeline / kanban view for opportunitiesCritical
Opportunities only have a table view. Every CRM shows a drag-and-drop kanban by sales stage (Lead → Qualified → Proposal → Closed). This is the primary way sales teams manage pipeline. A table alone is a CRM anti-pattern for opportunities.
No inline activity / timeline on recordsCritical
Contact and org detail pages show related records but no chronological activity feed (calls, emails, notes, status changes). The activity log exists as a global admin view only. CRM users expect a per-record timeline showing everything that happened to that record.
High priority — significant UX impact
Empty state pagesHigh
Sales page and Finance page are `Widget A / Widget B` placeholders. These are visible to all users. Should show summary metrics or redirect properly until built out.
No notes / comments on recordsHigh
No way to add free-text notes to a contact, org, or opportunity. This is a daily-use CRM feature. Without it users default to external tools, defeating the platform.
Opportunities table is flatHigh
No visual indication of deal health — no due date urgency coloring, no stage progress bar, no value totals in the header. Pipeline totals by stage belong above the table.
Dialog max-width inconsistencyHigh
Create Contact is `max-w-2xl`, Create Org is `max-w-xl`, IO dialog is `max-w-3xl`, User dialog is `max-w-xl`. No shared size token. Pick two sizes: standard (xl) and large (2xl/3xl) and apply consistently.
No global searchHigh
There is no cross-module search bar. Users cannot type "Acme" and see matching contacts, orgs, and opportunities together. This is essential for a multi-entity CRM.
Recycle bin UX is confusingHigh
Recycle bin is in the Admin section, not near the modules it serves. Deleted records are invisible until you navigate admin. Should surface a "deleted recently" indicator near each module or inside the filter bar.
Medium priority — polish and efficiency
Finance page is a dead endMedium
  • No IO summary metrics (total value, by status, overdue count)
  • No chart showing IO trends over time
  • Widget A / B placeholders are user-facing
Table loading statesMedium
  • Loading row shows only "Loading data..." text — no skeleton rows
  • Pagination shows stale numbers during refetch
  • No optimistic UI on status changes
Breadcrumb / back navMedium
Detail pages use plain text "Back to Contacts" links. No breadcrumb component. Contact → Org navigation loses context when traversing relationships.
No keyboard shortcutsMedium
No cmd+K global command palette. No keyboard navigation for tables. Power users in CRMs spend 80% of time in keyboard flow — this is a significant productivity gap.
Form validation UXMedium
Errors surface as a single red block at top of dialogs. No per-field inline validation. Required field markers exist but no real-time feedback until submit.
