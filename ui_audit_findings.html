Minor UI fixes
GridLayout on Finance & Sales dashboard pages is broken
finance/page.tsx and sales/page.tsx pass a gridConfig prop that doesn't exist on react-grid-layout. The correct prop names are cols and rowHeight directly on the component, not nested.
dashboard/finance/page.tsx · dashboard/sales/page.tsx
Replace gridConfig={{ cols: 12, rowHeight: 100 }} with cols={12} rowHeight={100}, and add the required layout={[]} prop or the grid won't render anything.
medium
Opportunities page pageSize ignored in Pagination display
The pageSize prop is computed as rangeStart && rangeEnd ? rangeEnd - rangeStart + 1 : 0 — this recalculates from server data each render and won't match the user's selected size on intermediate renders, causing the items-per-page control to flicker.
dashboard/sales/opportunities/page.tsx
Keep a local pageSize state (default 10), pass it to both useOpportunities and <Pagination>, and wire onPageSizeChange to update it.
medium
Switch component on Modules page leaks Radix props to DOM
The Modules page uses the custom <Switch> which spreads all props onto a motion.button. onCheckedChange is not a valid HTML attribute and will produce a React warning in the console.
dashboard/modules/page.tsx · components/ui/switch.tsx
The switch already destructures Radix-specific props — confirm onCheckedChange is in the destructured list and not forwarded via ...buttonProps.
medium
Background job polling never stops if dialog is closed mid-poll
In ModuleImportExportControls, the useEffect interval polls as long as status is "queued" or "running". If the user closes the dialog with resetImportState(), the job ID is cleared but the interval may fire one more tick and try to call onImportSuccess after unmount.
components/ui/ModuleImportExportControls.tsx
The cancelled flag in the effect already handles this — but confirm that resetImportState sets importJobStatus to something other than "queued"/"running" so the effect cleans up properly on the next render cycle.
